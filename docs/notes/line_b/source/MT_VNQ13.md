Q : P-NEW-28 — M1 "BUY 1회" 원칙과 5등분 슬라이싱의 관계 입니다.

  문서에 "BUY는 1회만 제출" 문구가 있는데, 5등분 슬라이싱이 확정되면서 해석이
  모호해졌습니다:

  - 해석 A: "1회" = Fill Window 전체 1번 (= 최대 5슬라이스 순차 실행) →
  슬라이싱과 호환됨
  - 해석 B: "1회" = 주문 1개 → 슬라이싱(최대 5개 주문)과 충돌

  현재는 ❓ 미확인 상태로 등록만 되어 있고, 사용자 확정이 필요합니다.

A : BUY 1회는 “Fill Window(240초) 1회 동안 수행되는 5-slice(20%×최대5) 순차 LIMIT 제출 시퀀스 1회”를 의미한다.
•	따라서 BUY 1회 내에서 최대 5개의 주문 제출은 허용되며, 이는 “BUY 1회” 원칙 위반이 아니다.
Cancel/TTL 규칙도 지금 정리대로 확정 가능
•	슬라이스별 TTL(30초) 만료 시 미체결 잔량만 취소 → 다음 슬라이스 진행
•	Fill Window 종료 시 열려있는 주문(open orders)만 취소, 체결분은 유지(보유)
•	CANCEL은 단일 로직에서만 수행(중복 타이머 금지)
체크(이것까지 적으면 완전 무결)
•	“슬라이스 간 동시 주문 금지(항상 0~1개 open order만 존재)”
